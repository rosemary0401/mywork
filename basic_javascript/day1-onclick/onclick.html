External JavaScript

1.JavaScript files have the file extension .js.

2.To use an external script, put the name of the script file in the src (source) attribute of a script tag

3.External scripts cannot contain script tags.

4.You can place an external script reference in head or body as you like.

5.To add several script files to one page - use several script tags

6.External scripts can be referenced with a full URL or with a path relative to the current web page.

總結:head、body、external file都可以放，一次可以放多個script tags

variables

Start the statement with var and separate the variables by comma

To declare all variables at the beginning of a script.

variables are often declared without a value. The value can be something that has to be calculated, or something that will be provided later, like user input.

A variable declared without a value will have the value undefined.

If you re-declare a JavaScript variable, it will not lose its value before you give another value to it.

If you put a number in quotes, the rest of the numbers will be treated as strings, and concatenated.

different：var x = "5" + 2 + 3 will display 523(string在前，數字在後者不會自動計算), but var x = 2 + 3 + "5" will display 55(數字在前者會自動計算)

If you add a number and a string, the result will be a string! Different sequences can produce different results.

Objects

1.objects can contain many values.

2.values are written as name:value pairs

3.These name:value pairs are properties or method. ex:car.color or car.start

4.Two ways to access object properties：objectName.propertyName or objectName["propertyName"]
ex:car.color or car["color"]

5.The property values of objects can also have methods also called functions stored as a property.
ex:var person{
firstName:"Rose"
lastName:"Chao"
fullName : function() {
return this.firstName + " " + this.lastName;
}}

About "this" keyword：this.firstName means that the firstName property of "this" object which refers to the "person" object.

6.To access object:objectName.methodName(), ex: person.fullName() If you access a method without parentheses()，it will return the function definition not a result of function.

7.When a JavaScript variable is declared with the keyword "new", the variable is created as an object
Don't declare strings, numbers, and booleans as objects.It slows down execution speed and produce some unexpected results.
ex:var x = new String(); // Declares x as a String object

8.Comparing two JavaScript objects will always return false.

Events

1.HTML events are "things" that happen to HTML elements.

2.When JavaScript is used in HTML pages, JavaScript can "react" on these events.

3.An HTML event can be something the browser does, or something a user does.

4.HTML allows event handler attributes, with JavaScript code, to be added to HTML elements.

ex: button onclick="document.getElementById('').innerHTML = Date()">The time is? /button>

or change the content of its own element : button onclick="this.innerHTML=Date()">The time is? /button>

5.可以在HTML的element掛handler，call function等

String

1.difference between indexOf(), and lastIndexOf() ：indexOf()指第一個指定的字符出現的位置， 而lastIndexOf()是找出指定字符最後出現的位置，若該字符重複多次出現，就會給出最後出現的位置

2.Both indexOf(), and lastIndexOf() return -1 if the text is not found.

3.兩者皆可放置第二個參數在()內，例如indeOf('John',5)，表示從左0數到第5的位置，開始往後搜尋'John'的從左到右第一個出現的位置，數字順序仍接續下去; lastIndexOf('John',5)表示從左0數到第5的位置，開始往前搜尋'John'最後出現的位置，順序從左到右找到字首J為最後出現的位置

4.search() is not equal to indexOf()

5.All string methods return a new string. They don't modify the original string.
Formally said: Strings are immutable: Strings cannot be changed, only replaced.
